# Settings Frontend

## Propósito

`settings-front` encapsula el acceso cliente al módulo y sus mensajes traducibles.

## Secciones

- `factory`: selección de provider por transporte.
- `i18n`: vocabulario del módulo y permisos.
- `interfaces`: contrato del provider.
- `providers`: implementaciones GraphQL y REST.

## Cuándo usarlo

Usalo cuando una app necesite consultar settings visibles o actualizar sus valores.
